# Documentation consolidation status

This `docs/` directory consolidates documentation across the repository into a single, browsable index without moving the original files.

- Source of truth: Original Markdown files remain in their respective project folders.
- How it works: `docs/README.md` is generated by `docs/build_index.py` and links back to the original documents.
- Why links instead of copies: This avoids duplication and keeps the documentation close to the code it describes while providing a central entry point.

## Update the index

Run the generator from the repository root or from this folder:

```bash
# From repo root
python3 docs/build_index.py

# Or from docs/
python3 build_index.py
```

The script will scan the repository for `*.md` files (excluding `docs/` and common build/cache folders) and regenerate `docs/README.md`.

## Notes

- Some documents may be outdated. The index groups files by top-level package to make review easier.
- If you rename or remove documents, rerun the generator to refresh the index.
- If you prefer a static copy under `docs/`, we can extend the script to copy files instead of linking.
